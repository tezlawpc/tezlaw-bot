    How filing actually works

    Read before assuming integration exists

    There is no public API for filing documents into the federal courts. The Administrative Office of the U.S. Courts exposes only the PACER Case Locator search API and an XML upload spec for bankruptcy petitions. Every motion, brief, complaint, and exhibit in district court still requires a human to log into the court's CM/ECF web interface and click through the docket events. The links below are the shortest path to that interface for each court you commonly practice in.

    For each court you intend to file in, you must hold an active PACER account, separately register as an e-filer with that court, and be admitted to practice before that bench. NextGen courts let you link PACER and filer credentials; CurrentGen still uses separate logins.

    Sync to Outlook / Google Calendar / Apple Calendar

    An .ics file with all deadline(s) has been downloaded. Each event includes the citation, party, and case notes in the body, plus two reminders (7 days and 1 day prior).

    Outlook (Desktop)
    1. Open the downloaded .ics file. Outlook will prompt to import it.
    2. Or in Outlook: File → Open & Export → Import/Export → Import an iCalendar file.
    3. Choose Import (events merge into your default calendar) or Open as New (separate read-only calendar). Import is usually what you want.
    Outlook (Web · outlook.com / Microsoft 365)
    1. Go to Calendar → Add calendar → Upload from file.
    2. Select the .ics file and choose which calendar to merge into.
    Google Calendar
    1. Open Google Calendar in a browser.
    2. Settings (⚙) → Import & export → Import.
    3. Select the .ics file and the target calendar.
    Apple Calendar (macOS)
    1. Double-click the .ics file. Calendar prompts for the target calendar; click OK.

    Re-import after schedule changes — events with the same UID will update, not duplicate.
